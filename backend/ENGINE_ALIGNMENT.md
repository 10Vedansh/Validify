# AI Engine — Frontend Compatibility Verification

## Output Shape Alignment

The orchestrator's `ValidationReport` is consumed by `validation.controller.ts` which maps it into the Prisma `Report` record. The frontend `Report` type reads from the Prisma model via API responses.

### Frontend `Report` → Orchestrator `ValidationReport` → Prisma `Report`

| Frontend `Report` field | Orchestrator source | Prisma column | Status |
|---|---|---|---|
| `id: string` | Generated by Prisma | `id` | ✅ |
| `ideaId: string` | From request param | `idea_id` | ✅ |
| `title: string` | `"${idea.name} — Validation Report"` | `title` | ✅ |
| `summary: string` | LLM-generated or fallback | `summary` | ✅ |
| `industry: string` | From Idea record | `industry` | ✅ |
| `score: ValidationScore` | `report.score` from orchestrator | `overall_score`, `market_score`, etc. | ✅ |
| `swot: SWOT` | `report.swot` from orchestrator | `strengths[]`, `weaknesses[]`, etc. | ✅ |
| `competitors: Competitor[]` | `report.competitors` | `competitors` (JSONB) | ✅ |
| `roadmap: RoadmapItem[]` | `report.roadmap` | `roadmap` (JSONB) | ✅ |
| `createdAt: string` | Prisma default | `created_at` | ✅ |

### ValidationScore mapping (7 dimensions)

| Dimension | Service | Prompt |
|---|---|---|
| `overall: number` (0-100) | `validation-score.service` | Weighted average of all dimensions |
| `market: number` (0-100) | `validation-score.service` | Market size, growth, urgency scoring |
| `team: number` (0-100) | `validation-score.service` | Founder-market fit assessment |
| `moat: number` (0-100) | `validation-score.service` | Defensibility, IP, network effects |
| `monetization: number` (0-100) | `validation-score.service` | Pricing model clarity, margins |
| `traction: number` (0-100) | `validation-score.service` | Evidence of demand, revenue |
| `risk: number` (0-100) | `validation-score.service` | Inverted — lower = more risky |

### SWOT mapping (4 arrays of strings)

| Array | Service | Prompt section |
|---|---|---|
| `strengths: string[]` | `swot.service` | Internal positives (3-5 items) |
| `weaknesses: string[]` | `swot.service` | Internal negatives (3-5 items) |
| `opportunities: string[]` | `swot.service` | External positives (3-5 items) |
| `threats: string[]` | `swot.service` | External negatives (3-5 items) |

## API Contract

### POST /ideas/:id/validate

**Request:** (authenticated, no body — validations are always for the full idea)

**Response (201):**
```json
{
  "validationId": "val_abc123",
  "reportId": "rpt_abc123",
  "score": {
    "overall": 82,
    "market": 86,
    "team": 72,
    "moat": 78,
```

**Response (200):**

## Frontend Service Compatibility

The frontend `ideasService` at `validifyai-main/src/services/ideas.service.ts` calls:

```typescript
async validate(ideaId: string): Promise<Validation> {
  const { data } = await api.post<Validation>(`/ideas/${ideaId}/validate`);
  return data;
}
```

The backend returns `{ validationId, reportId, score }`. The frontend `Validation` type expects:

```typescript
type Validation = {
  id: string;
  ideaId: string;
  score: ValidationScore;
  status: "draft" | "running" | "complete" | "failed";
  createdAt: string;
};
```

**Note:** The response shape from `POST /ideas/:id/validate` returns `{ validationId, reportId, score }` on success (when validation completes synchronously). For the frontend to consume this correctly, the `ideas.service.ts` on the frontend should be updated to extract the `score` and map `validationId` → `id`. Alternatively, the controller response can be adjusted to match the exact `Validation` type.

The `GET /validations/:id` endpoint returns the full `Validation` shape.

## Error Recovery Strategy

| Scenario | Handling |
|---|---|
| Single service failure (e.g. persona) | Other 6 continue; fallback data used for failed service |
| All services fail | Scores default to 50/50/50/50/50/50/50; SWOT empty; summary generated from idea data |
| OpenRouter timeout (60s) | Retry 3x with exponential backoff; then fallback model |
| OpenRouter 4xx (bad request) | No retry — thrown immediately |
| OpenRouter 429 (rate limit) | Retried with backoff like server errors |
| Summary generation fails | Template-based fallback using available scores and SWOT |
| Roadmap generation fails | Hard-coded 4-quarter fallback roadmap |
| Prisma write fails | Validation marked as FAILED with error message |
